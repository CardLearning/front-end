# API Contracts

API-контракты в проекте определяются в рамках Frontend-first development (FFD).

Контракт появляется после проектирования UI, TypeScript-типов, fixtures/mock data и первичной frontend-реализации. Backend должен реализовываться под уже проверенный пользовательский сценарий и согласованную структуру данных.

## Contract Workflow

1. Frontend описывает TypeScript-типы для данных экрана.
2. Frontend создает fixtures/mock data под эти типы.
3. UI реализуется и проверяется на fixtures.
4. По итогам UI формируется API-контракт.
5. Backend реализует контракт.
6. Frontend заменяет fixtures на реальные запросы.

## Contract Requirements

- Контракт должен покрывать данные, реально нужные интерфейсу.
- Контракт не должен заставлять frontend подстраиваться под преждевременную backend-модель.
- Изменения контракта должны сопровождаться обновлением TypeScript-типов и fixtures.
