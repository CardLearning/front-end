# Frontend Architecture

## Stack

Frontend-часть приложения разрабатывается на Next.js с использованием TypeScript.

Next.js является основным frontend-фреймворком проекта. Новые frontend-решения должны проектироваться с учетом маршрутизации Next.js, компонентной модели React и типизированной разработки на TypeScript.

## FSD Architecture

На frontend используется FSD-архитектура (Feature-Sliced Design).

Новые frontend-модули должны раскладываться по FSD-слоям и не должны нарушать направление зависимостей между слоями. Если для задачи нужен новый компонент, модель, fixture или helper, его место выбирается по FSD-смыслу, а не только по техническому типу файла.

## Development Methodology

Для frontend используется методология Frontend-first development (FFD).

Сначала проектируется и реализуется пользовательский интерфейс на fixtures/mock data. Backend реализуется после того, как интерфейс, TypeScript-типы и API-контракты достаточно понятны и согласованы.

Обязательный порядок работы:

1. Проектирование UI.
2. Создание TypeScript типов.
3. Создание fixtures/mock data.
4. Разработка frontend.
5. Определение API-контрактов.
6. Реализация backend.
7. Замена fixtures на реальные запросы.

## Practical Rules

- Frontend должен быть работоспособен на fixtures до подключения backend.
- TypeScript-типы создаются до основной реализации UI.
- API-контракты выводятся из потребностей готового интерфейса, а не наоборот.
- Реальные запросы подключаются после стабилизации UI, типов и контрактов.
- Если backend еще не готов, frontend-задача не должна блокироваться: используется mock data слой.
- Для актуальной документации по Next.js, React, Sass, TypeScript, FSD tooling и другим библиотекам используется Context7.
- Если Context7 не находит нужную документацию, это нужно явно указать и использовать лучший доступный первичный источник.

## Sass Modules: styles.module.scss

Для стилей используется Sass и модульные стили.

Правила для компонентов:

- файл стилей компонента всегда называется `styles.module.scss`;
- файл `styles.module.scss` должен лежать рядом с `.tsx` компонентом, к которому относится;
- стили подключаются в компоненте через `import s from "./styles.module.scss"`;
- классы из модуля используются через объект `s`;
- глобальные стили допускаются только для действительно глобальных базовых правил приложения.

Пример структуры:

```text
ComponentName/
  ComponentName.tsx
  styles.module.scss
```

Пример импорта:

```tsx
import s from "./styles.module.scss"
```
