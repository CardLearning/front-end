# Frontend Architecture

## Stack

Frontend-часть приложения разрабатывается на Next.js с использованием TypeScript.

Next.js является основным frontend-фреймворком проекта. Новые frontend-решения должны проектироваться с учетом маршрутизации Next.js, компонентной модели React и типизированной разработки на TypeScript.

## Development Methodology

Для frontend используется методология Frontend-first development (FFD).

Сначала проектируется и реализуется пользовательский интерфейс на fixtures/mock data. Backend реализуется после того, как интерфейс, TypeScript-типы и API-контракты достаточно понятны и согласованы.

Обязательный порядок работы:

1. Проектирование UI.
2. Создание TypeScript типов.
3. Создание fixtures/mock data.
4. Разработка frontend.
5. Определение API-контрактов.
6. Реализация backend.
7. Замена fixtures на реальные запросы.

## Practical Rules

- Frontend должен быть работоспособен на fixtures до подключения backend.
- TypeScript-типы создаются до основной реализации UI.
- API-контракты выводятся из потребностей готового интерфейса, а не наоборот.
- Реальные запросы подключаются после стабилизации UI, типов и контрактов.
- Если backend еще не готов, frontend-задача не должна блокироваться: используется mock data слой.
